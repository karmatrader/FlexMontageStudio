# CLAUDE.md

Этот файл предоставляет инструкции для Claude Code (claude.ai/code) при работе с кодом в данном репозитории.

## Обзор проекта

FlexMontage Studio — это приложение для автоматического AI-монтажа видео на Python с графическим интерфейсом PySide6. Оно автоматизирует редактирование видео, объединяя обработку аудио, AI-синтез голоса, генерацию субтитров и видеоэффекты. Приложение поддерживает работу с несколькими YouTube-каналами и обеспечивает пакетную обработку.

## Команды сборки и разработки

### Настройка окружения
```bash
# Настройка окружения и создание конфигурационных файлов
python setup_env.py                    # Создать .env с ключами по умолчанию
python setup_env.py --new-keys        # Сгенерировать новые ключи безопасности
python setup_env.py --production      # Подготовить для продакшена
python setup_env.py --check           # Проверить текущие настройки
```

### Сборка исполняемых файлов
```bash
# Сборка для macOS
./build_macos.sh                      # Создает .app пакет в dist/

# Сборка для Windows  
build_windows.bat                     # Создает .exe и .dist папку в dist/
```

### Запуск приложения
```bash
# Основные точки входа
python startup.py                     # Рекомендуемая точка входа
python main.py                        # Альтернативная точка входа

# Утилиты для разработки
python nuitka_command_generator.py    # Генерация оптимизированных команд сборки Nuitka
```

## Обзор архитектуры

### Основные компоненты
- **startup.py**: Главная точка входа с проверкой лицензий и инициализацией GUI
- **main.py**: Альтернативная точка входа и основная логика приложения
- **core/**: Основные модули приложения
  - `config_manager.py`: Управление конфигурацией каналов с интеграцией File API
  - `license_manager.py`: Проверка лицензий и шифрование
  - `task_manager.py`: Координация фоновых задач
  - `logging_config.py`: Централизованная конфигурация логирования

### UI слой
- **ui/main_window.py**: Основное окно GUI с вкладками
- **ui/**: UI компоненты и виджеты
  - `config_widgets.py`: Виджеты конфигурационных форм
  - `voice_selector_widget.py`: Интерфейс выбора голоса
  - `worker_threads.py`: Потоки фоновой обработки
  - `dialog_widgets.py`: Диалоговые окна и всплывающие окна

### Модули обработки
- **audio_processing.py**: Обработка аудио и интеграция синтеза голоса
- **video_processing.py**: Видеоэффекты, переходы и рендеринг
- **subtitles_processing.py**: Генерация субтитров и наложение
- **final_assembly.py**: Компиляция и экспорт видео
- **parallel_montage_manager.py**: Координация пакетной обработки

### Система конфигурации
- **config/env_manager.py**: Управление переменными окружения и ключами безопасности
- **channels.json**: Настройки каналов (создается автоматически)
- **license.json**: Данные лицензий (создается автоматически)
- **.env**: Ключи безопасности и переменные окружения

## Важные заметки по разработке

### Паттерны структуры файлов
- Конфигурационные файлы создаются автоматически при первом запуске и хранятся внешне (не в пакете)
- Приложение использует паттерн File API (`core/file_api.py`) для доступа к конфигурации
- UI виджеты модульные и могут импортироваться независимо
- Модули обработки предназначены для параллельного выполнения

### Система лицензирования
- Использует зашифрованное лицензирование с HMAC валидацией
- Демо-лицензия встроена и активируется автоматически
- Продакшн сборки требуют отдельного развертывания файла лицензии
- Ключи лицензий генерируются через `setup_env.py --new-keys`

### Система сборки
- Использует Nuitka для компиляции с обширными правилами исключения
- Статически включает FFmpeg для кроссплатформенной совместимости
- Каталог TestChannel включен для примера структуры данных
- Конфигурационные файлы намеренно исключены из сборок

### Обработка голоса
- Интегрируется с ElevenLabs API для синтеза голоса
- Локально кэширует модели голоса
- Поддерживает пакетную генерацию голоса для нескольких каналов
- API ключи хранятся в формате TestChannel/api_keys.csv

### Обработка ошибок
- Глобальный обработчик исключений в startup.py перехватывает все необработанные ошибки
- Комплексное логирование во всех модулях
- Плавная деградация при отсутствии опциональных зависимостей
